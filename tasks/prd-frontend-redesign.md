# PRD: LyingLLM 中文上帝视角前端重构

## 目标

基于 mind.txt 设计理念，将前端重构为面向中文用户的**上帝视角**狼人杀观战系统，支持日夜视觉模式、12座位环绕布局、思维过程面板、历史回放。

## 用户故事

### US-101: Setup 配置页（中文）
- 12 座位独立配置 provider/model
- 一键填充全部座位
- 校验按钮，显示中文错误/警告
- 开始游戏按钮（校验通过方可点击）
- 导航到 History 页面

### US-102: Game 观战页（上帝视角）
- **顶部栏**: 游戏标题、当前轮次、阶段名称、存活/死亡计数、日夜模式切换按钮
- **12 座位环绕布局**: 左右各 6 个座位卡片，中央为信息展示区
- **座位卡片**: 编号、角色图标+中文名、存活状态（亮色/暗色/死亡灰）、当前状态标签（思考中/发言中/已行动/闲置）
- **中央信息区**:
  - 公开发言（气泡形式）
  - 思维过程（可折叠，区分 summary/full/self_explanation）
  - 投票结果
  - 夜间行动详情（上帝视角标记"仅XX可见"）
- **底部时间线**: 横向时间轴，标注各阶段节点
- **日夜模式**: 随游戏阶段自动切换（night 阶段=夜间模式，day 阶段=日间模式），也可手动锁定

### US-103: History 历史页
- 列出已完成游戏（game_id、轮次、胜负方、结束时间）
- 点击可回放（复用 Game 页面，只读模式）

### US-104: 后端补充
- runner 在每次 agent 调用后写入 `reasoning_trace` 事件
- `GET /api/games` 返回所有游戏列表
- `GET /api/games/{id}` 返回完整玩家信息（含角色分配）

## 非目标

- 不在前端做规则判定
- 不接入真实 LLM（仍用 Mock）
- 不做复杂动画

## 技术要点

- 日夜模式用 CSS 变量 + React Context 管理
- 角色中文映射在前端做（role_id → 中文名 + emoji）
- 环绕布局用 CSS Grid 或绝对定位
- 思维面板默认折叠，点击展开
