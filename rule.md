# LyingLLM 狼人杀规则 v1

本规则用于 12 人标准局，不支持动态人数。它同时面向三类读者：

- 人类：可以直接理解一局游戏如何进行。
- 系统：可以据此实现状态机、动作校验、结算和日志。
- 大模型 Agent：可以据此理解自己的可见信息、行动目标和输出边界。

## 1. 固定配置

### 1.1 玩家与角色

总人数固定为 12 人，玩家编号固定为 `1` 到 `12`。

| 阵营     | 角色     | 数量 | role_id             |
| -------- | -------- | ---- | ------------------- |
| 好人阵营 | 预言家   | 1    | `seer`            |
| 好人阵营 | 女巫     | 1    | `witch`           |
| 好人阵营 | 猎人     | 1    | `hunter`          |
| 好人阵营 | 守卫     | 1    | `guard`           |
| 好人阵营 | 村民     | 4    | `villager`        |
| 狼人阵营 | 普通狼人 | 3    | `werewolf`        |
| 狼人阵营 | 白狼王   | 1    | `white_wolf_king` |

好人阵营共 8 人：4 神职 + 4 村民。
狼人阵营共 4 人：3 普通狼人 + 1 白狼王。

### 1.2 胜利条件

系统只根据真实角色和存活状态判定胜负，不根据玩家发言判定。

| 胜利方   | 条件                         |
| -------- | ---------------------------- |
| 好人阵营 | 所有狼人阵营玩家出局         |
| 狼人阵营 | 所有神职出局，或所有村民出局 |

胜利判定规则：

- `alive_gods == 0` 时，狼人胜利。
- `alive_villagers == 0` 时，狼人胜利。
- `alive_wolves == 0` 时，好人胜利。

变量定义：

- `alive_wolves`：所有存活的狼人阵营玩家数量，包含 `werewolf` 和 `white_wolf_king`。
- `alive_gods`：所有存活神职数量，包含 `seer`、`witch`、`hunter`、`guard`。
- `alive_villagers`：所有存活村民数量，只包含 `villager`。

若同一次夜间结算中同时满足“狼人屠边”和“所有狼人出局”，执行“狼刀在先”原则，狼人阵营优先获胜。

## 2. 信息可见性

### 2.1 公共信息

所有存活玩家和观赛者可见：

- 白天公开发言。
- 上警、退水、警长投票和警长结果。
- 白天投票票形和放逐结果。
- 公开死亡公告。
- 遗言。
- 白狼王自爆及其带走目标。

### 2.2 私有信息

只对指定玩家和观赛者可见：

- 玩家自己的真实身份。
- 预言家的查验结果。
- 女巫的药品状态和可见刀口。
- 守卫上一晚守护目标。
- 猎人当前是否可以开枪。

### 2.3 狼人阵营信息

只对狼人阵营玩家和观赛者可见：

- 狼人同伴身份。
- 夜间狼人讨论内容。
- 狼人最终击杀目标。

### 2.4 Agent 禁止项

Agent 只能依据自己可见的信息行动。

- 不得读取或使用不可见身份、夜间行动、其他玩家私有结果。
- 不得把自己的不可公开信息当作系统事实强行写入公共日志。
- 可以在发言中撒谎、伪装、诈身份，但这只是一种发言行为，不改变真实状态。

## 3. 角色技能

### 3.1 预言家 `seer`

阵营：好人阵营，神职。

夜间技能：

- 每晚选择 1 名存活玩家查验。
- 查验结果只有两种：`wolf` 或 `not_wolf`。
- 预言家只能知道目标是否为狼人阵营，不能知道具体角色。

限制：

- 不能查验已出局玩家。
- 可以查验自己以外任意存活玩家。
- 查验结果只写入预言家私有记忆。

### 3.2 女巫 `witch`

阵营：好人阵营，神职。

药品：

- 解药 1 瓶，全局最多使用一次。
- 毒药 1 瓶，全局最多使用一次。

夜间技能：

- 若解药未使用，系统告知女巫当晚狼人击杀目标。
- 女巫可以选择是否使用解药救该目标。
- 女巫可以选择是否使用毒药毒杀 1 名存活玩家。

限制：

- 女巫可以自救。
- 同一夜不能同时使用解药和毒药。
- 解药使用后，后续夜晚女巫不再获得狼人击杀目标信息。
- 毒药只能作用于 1 名存活玩家。
- 被女巫毒杀的猎人不能开枪。

同守同救：

- 若同一晚同一名玩家既被守卫守护，又被女巫使用解药救起，该玩家死亡。

### 3.3 猎人 `hunter`

阵营：好人阵营，神职。

死亡触发技能：

- 猎人出局时，若当前状态允许开枪，可以选择带走 1 名存活玩家。
- 猎人也可以选择不开枪。

能否开枪：

| 死因           | 是否可开枪 |
| -------------- | ---------- |
| 狼人击杀       | 可以       |
| 白天投票放逐   | 可以       |
| 白狼王自爆带走 | 可以       |
| 女巫毒杀       | 不可以     |

猎人开枪造成的死亡会立即加入当前死亡队列，并在之后进行胜负判定。

### 3.4 守卫 `guard`

阵营：好人阵营，神职。

夜间技能：

- 每晚可以选择 1 名玩家守护。
- 被守护玩家若当晚被狼人击杀，则不会因狼刀死亡。

限制：

- 守卫可以守护自己。
- 守卫可以空守。
- 守卫不能连续两晚守护同一名玩家。
- 空守不算作守护某名玩家，不影响下一晚目标限制。

同守同救：

- 若守护目标也是女巫解药目标，该目标死亡。

### 3.5 村民 `villager`

阵营：好人阵营，村民。

村民没有夜间技能。村民通过白天发言、投票、分析票形帮助好人阵营找出狼人。

### 3.6 普通狼人 `werewolf`

阵营：狼人阵营。

夜间技能：

- 与所有狼人阵营玩家共同讨论。
- 每晚共同选择 1 名存活玩家作为击杀目标。

白天能力：

- 普通狼人可以自爆。
- 普通狼人自爆后自己可以发表遗言，遗言发表完毕后自己出局，当前白天环节立即结束，直接进入黑夜。
- 普通狼人自爆不能带走其他玩家。

### 3.7 白狼王 `white_wolf_king`

阵营：狼人阵营。

夜间能力：

- 与普通狼人一起参与夜间讨论和击杀。

白天技能：

- 白狼王可以在白天公开翻牌自爆。
- 自爆时，白狼王自己出局，并选择带走 1 名存活玩家。
- 白狼王自爆成功后，可以发表遗言（一般用于指挥夜间刀法，即“指刀”），随后被带走的玩家发表自己的遗言。当白狼王和被带走的人遗言和技能使用完毕后，当前白天环节立即结束，直接进入当晚黑夜。

限制：

- 白狼王只能在白天自爆带人。
- 白狼王不能在夜间发动自爆带人。
- 被白狼王带走的玩家如果有死亡触发技能，按其自身规则处理。例如猎人被带走且可开枪，则可以开枪。

遗言：

- 白狼王自爆后有遗言。
- 被白狼王带走的玩家视为白天死亡，有遗言。

## 4. 游戏阶段

### 4.1 阶段总览

固定流程如下：

```text
SETUP
→ ROLE_ASSIGNMENT
→ NIGHT_1
→ DAWN_1
→ SHERIFF_ELECTION
→ DAY_1
→ NIGHT_2
→ DAWN_2
→ DAY_2
→ ...
→ GAME_END
```

第一夜之后先公布死讯，再进入警长竞选，警长竞选完成后，再处理遗言和死亡技能。
警长竞选只在第一天进行。

### 4.2 夜晚行动顺序

每个夜晚严格按以下顺序执行：

1. 守卫行动。
2. 狼人讨论并确定击杀目标。
3. 女巫行动。
4. 预言家行动。
5. 系统统一结算夜间死亡。
6. 死亡触发技能处理。
7. 遗言资格判断。
8. 胜负判定。

夜间角色只提交行动意图，不能直接修改玩家生死。所有狼刀、守护、解药、毒药在统一结算阶段生效。

### 4.3 第一夜

第一夜与普通夜晚相同，但遗言规则不同：

- 第一夜死亡玩家有遗言。
- 第一夜之后的夜间死亡玩家没有遗言。

### 4.4 天亮阶段 `DAWN`

天亮时系统公布死亡结果：

- 公布死亡玩家编号。
- 不公布死因。
- 不公布死亡玩家身份，除非该死亡来自白天公开技能或规则另有要求。

夜间真实死因只进入观赛者日志和系统日志，不进入公共信息。

### 4.5 警长竞选

警长竞选只发生在第一天。

流程：

1. 所有玩家选择是否上警。
2. 上警玩家按系统指定顺序发言。
3. 上警玩家可以退水。退水后不再参与警长竞选。
4. 未上警且存活的玩家投票。
5. 得票最高的上警玩家当选警长。
6. 若最高票平票，平票候选人再次发言，未上警玩家重新投票。
7. 若重投后仍平票，本局无警长。

警长权利：

- 白天投票权重为 `1.5`。
- 可以决定白天发言顺序的起点和方向。
- 死亡时可以移交警徽给 1 名存活玩家，也可以撕毁警徽。

### 4.6 白天阶段

白天固定流程：

1. 处理上一个夜晚后的遗言。
2. 警长或系统确定发言顺序。
3. 存活玩家依次发言。
4. 玩家可在允许时自爆。
5. 若无人自爆，进入投票。
6. 统计票形并放逐得票最高者。
7. 处理放逐死亡技能。
8. 处理遗言。
9. 胜负判定。
10. 若游戏未结束，进入下一夜。

白天发言必须串行，后发言玩家可以看到先发言玩家的公开发言。

## 5. 投票规则

### 5.1 白天放逐投票

投票对象：

- 只能投给存活玩家。
- 允许弃票。

计票：

- 普通玩家票权为 `1.0`。
- 警长票权为 `1.5`。
- 弃票记录进票形日志，但不计入任何候选人得票。
- 得票最高的玩家被放逐。

平票：

1. 若两名或多名玩家并列最高票，进入平票发言。
2. 只有平票候选人发言。
3. 非平票候选人在平票候选人中重新投票。
4. 若重投后仍平票，本轮无人出局。

全员弃票：

- 本轮无人出局。
- 直接进入胜负判定。

### 5.2 警长投票

警长投票只由未上警玩家参与。

- 得票最高者当选警长。
- 平票后进行平票候选人发言和重投。
- 重投仍平票，则本局无警长。

## 6. 死亡、技能与遗言

### 6.1 死亡处理顺序

只要有玩家出局，系统按以下顺序处理：

1. 标记死亡玩家。
2. 判断并执行死亡触发技能。
3. 将死亡触发技能造成的新死亡加入队列。
4. 判断遗言资格。
5. 执行遗言。
6. 胜负判定。

若多个玩家同一批次死亡，按玩家编号从小到大处理。若死亡触发技能造成新死亡，新死亡追加到队列尾部。

### 6.2 遗言规则

| 死亡场景             | 是否有遗言 | 说明                           |
| -------------------- | ---------- | ------------------------------ |
| 第一夜死亡           | 有         | 无论死因，均有遗言             |
| 第二夜及之后夜间死亡 | 无         | 直接出局，无遗言               |
| 白天投票放逐         | 有         | 先处理死亡技能，再遗言         |
| 白天普通狼人自爆     | 有         | 自爆后发言，然后进入黑夜       |
| 白天白狼王自爆       | 有         | 白狼王有遗言，被带走者也有遗言 |
| 白天技能死亡         | 有         | 例如猎人开枪、白狼王带人       |

猎人夜晚被狼刀后开枪带走的目标，视为夜间死亡：

- 如果发生在第一夜，该目标有遗言。
- 如果发生在第二夜及之后，该目标无遗言。

猎人白天死亡后开枪带走的目标，视为白天死亡，有遗言。

## 7. 夜间统一结算

夜间结算输入：

```yaml
night_actions:
  guard_target: player_id | null
  wolf_kill_target: player_id
  witch_save_used: true | false
  witch_poison_target: player_id | null
  seer_check_target: player_id | null
```

结算规则：

1. 记录狼刀目标。
2. 记录守卫目标。
3. 记录女巫是否使用解药。
4. 记录女巫毒药目标。
5. 若狼刀目标被守卫守护，狼刀死亡被阻止。
6. 若狼刀目标被女巫解药救起，狼刀死亡被阻止。
7. 若同一玩家同时被守卫守护和女巫解药救起，该玩家死亡。
8. 若玩家被女巫毒药命中，该玩家死亡。
9. 若同一玩家同时有多个死因，只产生一次死亡，系统日志记录全部死因。
10. 预言家查验结果写入预言家私有信息，不影响死亡结算。

狼刀在先原则：

- 若同一夜晚的结算同时导致狼人屠边和所有狼人出局，狼人阵营优先获胜。

## 8. 自爆规则

### 8.1 普通狼人自爆

普通狼人可以在白天公开自爆。

效果：

- 普通狼人自己立即出局。
- 当前白天阶段立即结束。
- 不再继续当前发言或投票。
- 进入死亡技能、遗言和胜负判定。
- 若游戏未结束，直接进入黑夜。

普通狼人自爆不能带人。

### 8.2 白狼王自爆

白狼王可以在白天公开自爆并带走 1 名存活玩家。

效果：

- 白狼王自己立即出局。
- 被选择目标立即出局。
- 不再继续当前发言或投票。
- 处理死亡技能和遗言。
- 当前白天阶段立即结束。
- 胜负判定。
- 若游戏未结束，直接进入黑夜。

## 9. 大模型 Agent 行为规范

每个 Agent 每次行动必须只输出当前阶段允许的内容。

### 9.1 通用输出原则

- 输出必须是结构化 JSON。
- 不得输出系统未要求的动作。
- 不得选择非法目标。
- 不得引用不可见信息。
- 可以撒谎、伪装、诈身份，但必须以公开发言形式出现。

### 9.2 常用动作类型

```yaml
actions:
  guard:
    fields: [target]
    target: player_id | null
  wolf_vote_kill:
    fields: [target, reason]
    target: alive_player_id
  witch:
    fields: [use_save, poison_target]
    use_save: true | false
    poison_target: alive_player_id | null
  seer:
    fields: [target]
    target: alive_player_id
  speech:
    fields: [content]
  vote:
    fields: [target]
    target: alive_player_id | abstain
  hunter_shoot:
    fields: [target]
    target: alive_player_id | null
  sheriff_transfer:
    fields: [target]
    target: alive_player_id | tear_badge
  self_destruct:
    fields: [target]
    target: alive_player_id | null
```

### 9.3 非法动作处理

系统发现非法动作时：

1. 告知 Agent 错误原因。
2. 要求 Agent 重新输出。
3. 超过重试次数后，系统使用默认动作。

默认动作：

| 阶段     | 默认动作                   |
| -------- | -------------------------- |
| 守卫     | 空守                       |
| 狼人击杀 | 在合法存活好人中随机选择   |
| 女巫     | 不救、不毒                 |
| 预言家   | 在未查验存活玩家中随机选择 |
| 发言     | 空发言                     |
| 投票     | 弃票                       |
| 猎人     | 不开枪                     |
| 警徽移交 | 撕毁警徽                   |

## 10. 系统实现常量

```yaml
game:
  player_count: 12
  roles:
    seer: 1
    witch: 1
    hunter: 1
    guard: 1
    villager: 4
    werewolf: 3
    white_wolf_king: 1

win_condition:
  village: all_wolves_dead
  wolves: all_gods_dead_or_all_villagers_dead
  tie_breaker_at_night: wolf_kill_first
  wolf_roles: [werewolf, white_wolf_king]
  god_roles: [seer, witch, hunter, guard]
  villager_roles: [villager]

night_order:
  - guard
  - wolves
  - witch
  - seer

witch:
  can_self_save: true
  can_use_both_potions_same_night: false
  loses_kill_info_after_save_used: true
  poison_blocks_hunter_shot: true

guard:
  can_self_guard: true
  can_skip_guard: true
  cannot_guard_same_player_consecutively: true
  guard_and_save_same_target_dies: true

hunter:
  can_shoot_when_killed_by_wolves: true
  can_shoot_when_exiled: true
  can_shoot_when_taken_by_white_wolf_king: true
  can_shoot_when_poisoned: false

sheriff:
  enabled: true
  vote_weight: 1.5
  revote_if_tie: true
  no_sheriff_if_revote_tie: true
  can_transfer_badge_on_death: true

vote:
  exile_rule: highest_votes
  allow_abstain: true
  abstain_counts_as_candidate_vote: false
  revote_if_tie: true
  no_elimination_if_revote_tie: true

last_words:
  first_night_deaths: true
  night_deaths_after_first_night: false
  day_exile_deaths: true
  day_skill_deaths: true
  death_skill_before_last_words: true

self_destruct:
  normal_werewolf_can_self_destruct: true
  normal_werewolf_takes_player: false
  white_wolf_king_can_self_destruct: true
  white_wolf_king_takes_player: true
  self_destruct_ends_current_day: true
```
